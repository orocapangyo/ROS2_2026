# ROS2 프로그래밍 첫걸음 - 글쓰기 스타일 가이드 (Writing Style Guide)

이 문서는 『ROS2 프로그래밍 첫걸음』 교재의 일관된 톤앤매너와 문서 구조를 유지하기 위한 집필 가이드라인입니다. 새로운 장을 작성하거나 기존 초안을 편집할 때 이 가이드라인을 반드시 준수해야 합니다.

---

## 1. 톤앤매너 (Tone and Voice)

* **친절하고 정중한 설명조 (해요체 사용)**
  * 본문 문장 종결어미는 주로 **해요체**(`~입니다`, `~합니다`, `~있습니다`, `~됩니다`, `~봅니다`)를 사용합니다. 독자를 직접 가르치는 다정한 어조를 유지합니다.
  * **피해야 할 말투**: `~한다`, `~이다` 같은 평어(해라체)는 본문에서 사용하지 않습니다. 단, 정의 상자나 참고 표의 일부 설명에서는 예외적으로 plain style을 사용할 수 있습니다.
* **직관적인 비유와 예시 활용**
  * 복잡한 기술 개념(DDS, QoS, Topic 등)을 설명할 때 일상적인 비유를 먼저 제시합니다.
    * *예: 토픽 = 라디오 방송, 서비스 = 질문과 대답, 액션 = 택배 배송 추적*
* **친절하고 명확한 단계별 지시어**
  * 따라하기나 실습 단계는 `1단계. [행동]을 합니다.`, `2단계. [행동]을 합니다.` 형태로 정중하고 명확하게 서술합니다.

---

## 2. 용어 및 코드 표기 규칙

* **한글 번역어와 영문 병기**
  * 주요 기술 용어를 처음 언급할 때는 한글 번역어 뒤에 괄호로 영문을 병기합니다.
    * *예: 분산 발견(distributed discovery), 미들웨어(middleware), 배포판(distribution)*
* **코드 및 변수 강조 (Backticks)**
  * 명령어, 패키지 이름, 환경 변수, 파일명, 코드 클래스/함수명은 항상 백틱(`` ` ``)으로 감싸 강조합니다.
    * *예: `rclpy`, `setup.bash`, `ROS_DOMAIN_ID`, `talker`*
* **명령어 실행창과 결과 구분**
  * 명령어 블록은 `bash` 또는 `powershell` 지정을 확실히 하고, 실행 결과가 중요할 때는 텍스트 블록(`text`)으로 출력을 명시해 줍니다.

---

## 3. 시각 요소 및 커스텀 박스 포맷

* **그림(도해) 캡션**
  * 그림이 들어갈 위치에는 다음과 같이 명확한 캡션을 넣습니다.
    * `![그림 X-Y. [그림 설명]]()`
* **정의 박스 (Definition Callout)**
  * 용어에 대한 명확한 정의는 인용구(`>`)를 사용하여 표기하며, 정의의 내용은 plain style(`~다`)로 서술합니다.
    ```markdown
    > **정의(N) [용어명]**  
    > [용어에 대한 설명문 (~다로 끝남)]
    ```
* **칼럼 박스 (Column Callout)**
  * 본문 흐름 외의 흥미로운 역사나 배경 설명은 칼럼 상자로 분리합니다.
    ```markdown
    > **[칼럼] [칼럼 제목]**  
    > [칼럼 설명 본문 (~합니다로 서술)]
    ```
* **노트/경고 박스 (Note/Warning Callout)**
  * 독자가 주의해야 하거나 알아두면 좋은 팁은 인용 블록을 사용해 가독성을 높입니다.
    ```markdown
    > **생각해 보기**: [질문이나 미션 서술]
    ```

---

## 4. 장(Chapter)의 표준 구조

각 장은 반드시 아래의 논리적인 구조를 따릅니다.

1. **장 제목**: `# 제N장 [장 제목]` (예: `# 제1장 ROS2 개요와 개발 환경 구축`)
2. **이 장에서 다루는 내용**:
   * `### 이 장에서 다루는 내용` 이라는 소제목을 부여합니다.
   * 아래에 요약 불릿 리스트(`* ...`)를 작성하며, 문장 끝은 평어(해라체, `~특징`, `~방법` 또는 `~다`)로 간결하게 종결합니다.
3. **학습 목표**:
   * `### 학습 목표` 라는 소제목을 부여합니다.
   * `이 장의 학습 목표는 다음과 같습니다.` 라는 도입부 문장을 적습니다.
   * 아래에 번호 목록(`1. ...`, `2. ...`)을 작성하며, 문장 끝은 평어(해라체, `~할 수 있다.`, `~이해한다.`)로 종결합니다.
   
   *장 앞부분 블록 구성 예시:*
   ```markdown
   ### 이 장에서 다루는 내용
   * 서비스와 토픽의 통신 방식 차이 및 서비스의 특징
   * 서비스 서버와 클라이언트의 역할 및 구현 방법
   * C++로 서비스 통신 구현하기
   * ros2 service 명령어를 활용한 서비스 디버깅 방법

   ### 학습 목표
   이 장의 학습 목표는 다음과 같습니다.
   1. 서비스(service)의 요청–응답(request/response) 모델을 토픽과 구분하여 설명할 수 있다.
   2. rclpy/rclcpp로 서비스 서버·클라이언트를 직접 작성할 수 있다.
   ```
4. **본문 섹션**: `## N.M [섹션 제목]` (1.1, 1.2 등)
5. **정리**: `## 정리`로 주요 요약 전달.
6. **핵심 용어**: `## 핵심 용어`로 주요 용어와 한 줄 설명 제공.
7. **연습문제**: `## 연습문제`로 배운 내용을 스스로 점검할 수 있는 질문 리스트 제공.
