## 4.1 서비스 통신 모델

### 서비스란 — 한 번의 질문과 답
토픽이 라디오 방송(broadcast)처럼 정보를 일방적으로 내보내는 단방향 스트림 방식이라면, 서비스는 일대일로 통화하며 질문과 대답을 나누는 양방향 통신 방식입니다. 

서비스 통신 모델은 정보를 제공하는 **서버(server)** 와 정보를 요청하는 **클라이언트(client)** 로 나뉩니다. 클라이언트가 특정 연산이나 처리를 요청(request)하면, 서버는 그 요청을 처리한 뒤 결과를 응답(response)으로 돌려줍니다. 이러한 구조를 요청-응답 모델이라고 부르며, 요청 1회당 응답 1회가 매칭됩니다.

```text
[클라이언트] --요청(request)-->  [서버]
[클라이언트] <--응답(response)-- [서버]
```

토픽과 서비스는 다음과 같은 명확한 차이점을 가집니다.

| 구분 | 토픽(topic) | 서비스(service) |
| :--- | :--- | :--- |
| **방향** | 단방향 | 양방향(요청·응답) |
| **횟수** | 지속적인 데이터 흐름 | 호출할 때 1회 |
| **용도** | 센서 데이터 스트림, 속도 명령 등 지속 흐름 | 연산 요청, 상태 변경, 일회성 설정 등 |

표 4-1. 토픽과 서비스의 비교

### 언제 서비스를 쓰나
서비스는 "두 값을 더해줘", "현재 로봇의 위치가 표시된 지도를 저장해줘", "로봇의 LED를 켜거나 꺼줘"와 같이 **요청한 일의 수행 여부나 결과값을 즉각적으로 확인해야 할 때** 사용합니다. 

다만, 로봇에게 "저 방으로 이동해라"와 같이 작업 수행에 오랜 시간이 걸리고 중간 과정을 모니터링해야 하는 경우에는 5장에서 배울 **액션(action)** 을 사용하는 것이 적합합니다.

### 서비스 타입 — .srv 파일
서비스 통신을 하기 위해서는 어떤 데이터를 요청하고 응답할지 형식을 정의해야 합니다. 이 형식은 `.srv` 파일에 선언합니다. `.srv` 파일은 세 개의 하이픈(`---`)을 구분선으로 사용하여 윗부분은 요청(request) 필드, 아랫부분은 응답(response) 필드를 정의합니다.

이번 장 실습에서 사용할 기본 예제 인터페이스인 `example_interfaces/srv/AddTwoInts`는 다음과 같이 구성되어 있습니다.

```text
int64 a          # 요청(request) 데이터 1
int64 b          # 요청(request) 데이터 2
---
int64 sum        # 응답(response) 데이터
```

이 인터페이스 구조는 터미널 창에 다음 명령어를 입력하여 언제든지 확인할 수 있습니다.

```bash
ros2 interface show example_interfaces/srv/AddTwoInts
```

> **정의(1) 서비스(service)**  
> ROS 2에서 클라이언트 노드가 요청을 보내고 서버 노드가 이에 대한 응답을 반환하는 일대일 양방향 통신 방식이다.

## 4.2 서비스 서버와 클라이언트

파이썬(Python)의 `rclpy` 라이브러리를 사용해 두 정수를 더해주는 서비스 서버와 클라이언트를 구현해 보겠습니다.

### Python 서비스 서버 구현하기
`my_py_pkg/my_py_pkg/add_two_ints_server.py` 파일을 생성하고 아래 코드를 작성합니다.

```python
import rclpy
from rclpy.node import Node
from example_interfaces.srv import AddTwoInts


class AddTwoIntsServer(Node):
    def __init__(self):
        super().__init__("add_two_ints_server")
        # 서비스 서버 생성: 타입, 서비스명, 콜백 함수 지정
        self.server_ = self.create_service(
            AddTwoInts, "add_two_ints", self.callback_add)
        self.get_logger().info("서비스 서버 준비 완료")

    def callback_add(self, request, response):
        response.sum = request.a + request.b
        self.get_logger().info(f"{request.a} + {request.b} = {response.sum}")
        return response


def main(args=None):
    rclpy.init(args=args)
    node = AddTwoIntsServer()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()
```

이 노드를 실행하기 위해 `setup.py` 파일의 `console_scripts`에 아래와 같이 등록합니다.

```python
entry_points={
    'console_scripts': [
        'add_two_ints_server = my_py_pkg.add_two_ints_server:main',
    ],
},
```

이후 패키지를 빌드하고 실행해 둡니다.

```bash
cd ~/ros2_ws
colcon build --packages-select my_py_pkg
source install/setup.bash
ros2 run my_py_pkg add_two_ints_server
```

### Python 서비스 클라이언트 구현하기
서버가 준비되면 요청을 보낼 클라이언트 노드가 필요합니다. `my_py_pkg/my_py_pkg/add_two_ints_client.py` 파일을 생성하고 작성합니다.

```python
import rclpy
from rclpy.node import Node
from example_interfaces.srv import AddTwoInts


class AddTwoIntsClient(Node):
    def __init__(self):
        super().__init__("add_two_ints_client")
        # 서비스 클라이언트 생성: 타입, 서비스명 지정
        self.client_ = self.create_client(AddTwoInts, "add_two_ints")
        # 서비스 서버가 활성화될 때까지 1초 간격으로 대기
        while not self.client_.wait_for_service(timeout_sec=1.0):
            self.get_logger().warn("서버 대기 중...")
        self.call_add(3, 5)

    def call_add(self, a, b):
        request = AddTwoInts.Request()
        request.a = a
        request.b = b
        # 비동기 방식으로 서비스 호출
        future = self.client_.call_async(request)
        future.add_done_callback(self.callback_response)

    def callback_response(self, future):
        try:
            response = future.result()
            self.get_logger().info(f"결과: {response.sum}")
        except Exception as e:
            self.get_logger().error(f"서비스 호출 실패: {e}")


def main(args=None):
    rclpy.init(args=args)
    node = AddTwoIntsClient()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()
```

서버와 마찬가지로 `setup.py`에 클라이언트 노드 실행 항목을 등록합니다.

```python
'add_two_ints_client = my_py_pkg.add_two_ints_client:main',
```

빌드 후 새로운 터미널에서 클라이언트를 실행하면 로그에 `결과: 8`이 출력되는 것을 볼 수 있습니다.

```bash
ros2 run my_py_pkg add_two_ints_client
```

> ⚠️ **동기 호출의 함정 (Deadlock)**  
> `rclpy`에서 동기식 호출 메서드(`client.call(request)`)를 사용하거나 `rclpy.spin_until_future_complete`를 단일 스레드 콜백 내부에서 중첩으로 실행하면 프로그램이 정지해버리는 교착 상태(deadlock)가 발생할 가능성이 높습니다. 입문 단계에서는 항상 위 예제와 같이 `call_async` 메서드와 완료 콜백(`done_callback`)을 조합하는 비동기 패턴을 권장합니다.

## 4.3 C++로 구현하기

동일한 서비스 통신을 C++ 언어 기반의 `rclcpp` 라이브러리로 구현해 보겠습니다.

### C++ 서비스 서버 작성
`my_cpp_pkg/src/add_two_ints_server.cpp` 파일을 만들고 아래 코드를 작성합니다.

```cpp
#include "rclcpp/rclcpp.hpp"
#include "example_interfaces/srv/add_two_ints.hpp"

using std::placeholders::_1;
using std::placeholders::_2;

class AddTwoIntsServer : public rclcpp::Node {
public:
    AddTwoIntsServer() : Node("add_two_ints_server") {
        // 서비스 서버 생성: 타입, 서비스명, 콜백 함수(바인딩) 지정
        server_ = this->create_service<example_interfaces::srv::AddTwoInts>(
            "add_two_ints",
            std::bind(&AddTwoIntsServer::callbackAdd, this, _1, _2));
        RCLCPP_INFO(this->get_logger(), "C++ 서비스 서버 준비 완료");
    }

private:
    void callbackAdd(
        const example_interfaces::srv::AddTwoInts::Request::SharedPtr req,
        const example_interfaces::srv::AddTwoInts::Response::SharedPtr res) {
        res->sum = req->a + req->b;
        RCLCPP_INFO(this->get_logger(), "%ld + %ld = %ld", req->a, req->b, res->sum);
    }
    rclcpp::Service<example_interfaces::srv::AddTwoInts>::SharedPtr server_;
};

int main(int argc, char **argv) {
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<AddTwoIntsServer>());
    rclcpp::shutdown();
    return 0;
}
```

### CMakeLists.txt & package.xml 빌드 설정
C++ 빌드 관리를 위해 의존성을 추가해 주어야 합니다.

`CMakeLists.txt` 파일에 아래 내용을 추가합니다.

```cmake
find_package(rclcpp REQUIRED)
find_package(example_interfaces REQUIRED)

add_executable(add_two_ints_server src/add_two_ints_server.cpp)
ament_target_dependencies(add_two_ints_server rclcpp example_interfaces)

install(TARGETS add_two_ints_server
  DESTINATION lib/${PROJECT_NAME})
```

`package.xml` 파일의 의존성 영역에 아래 두 줄을 등록합니다.

```xml
<depend>rclcpp</depend>
<depend>example_interfaces</depend>
```

작성이 완료되면 패키지를 빌드하고 실행합니다.

```bash
cd ~/ros2_ws
colcon build --packages-select my_cpp_pkg
source install/setup.bash
ros2 run my_cpp_pkg add_two_ints_server
```

> **정의(2) RMW**  
> ROS 2와 DDS 사이를 연결하는 추상화 계층이다. 이 덕분에 C++ 노드와 Python 노드처럼 서로 다른 언어로 구현되더라도 RMW 계층 위에서 규격화된 메시지를 주고받으며 원활하게 소통할 수 있다.

## 4.4 ros2 service 명령으로 살펴보기

개발 도중 노드가 올바르게 서비스를 제공하고 통신을 맺고 있는지 터미널 상에서 확인할 수 있는 유용한 디버깅 명령어들입니다.

### 서비스 목록 확인
현재 활성화되어 있는 모든 서비스 목록을 보여줍니다.
```bash
ros2 service list
```

### 서비스 타입 확인
특정 서비스의 타입을 상세히 보여줍니다.
```bash
ros2 service type /add_two_ints
```

### 서비스 수동 호출 (중요)
코드 작성 전에 서비스 서버가 정상 동작하는지 테스트할 수 있습니다. 타입 뒤에 요청부 데이터를 JSON 형식의 문자열로 인자 전달합니다.
```bash
ros2 service call /add_two_ints example_interfaces/srv/AddTwoInts "{a: 10, b: 20}"
```

### 서비스 인터페이스 구조 확인
서비스가 정의하고 있는 요청/응답 변수 규격을 확인합니다.
```bash
ros2 interface show example_interfaces/srv/AddTwoInts
```

## 코드 해설 · 실행 확인 · 버전 체크

- **코드 해설 포인트**: `.srv` 서비스 파일의 요청부(`Request`)와 응답부(`Response`) 분리 규조를 바탕으로, 서버의 `create_service`, 클라이언트의 `create_client` 호출 및 비동기 호출 처리를 위한 `call_async`와 `done_callback` 결합 구조를 상세히 이해합니다.
- **실행 확인 포인트**: `ros2 service call`을 이용하여 Python 서버와 C++ 서버 각각에 요청을 보냈을 때 연산 결과가 터미널에 반환되는지 확인합니다. 또한 `ros2 service list`에 생성한 서비스명이 잡히는지 검증합니다.
- **버전/환경 체크**: ROS 2 Jazzy 기준의 비동기 Future 객체 반환 및 RMW Fast DDS 미들웨어 계층의 동작 확인 흐름을 기준으로 검토합니다.

## 트러블슈팅

| 증상 | 원인 | 해결 |
| :--- | :--- | :--- |
| 클라이언트가 켜진 채 영원히 대기함 | 서비스 서버 노드가 켜지지 않았거나 서비스명이 다름 | `ros2 service list`로 이름 대조 및 서버 기동 여부 확인 |
| `call_async` 호출 후 응답 콜백이 반응 없음 | 실행 노드 루프에 `spin`을 걸지 않았거나 콜백 누락 | `rclpy.spin(node)` 또는 `rclcpp::spin`이 돌고 있는지 점검 |
| 서비스 호출 도중 타입 에러가 발생함 | 요청 인자(JSON 구조 등) 포맷 및 필드 오타 | `ros2 interface show`로 필드명(`a`, `b`)을 재확인하고 규격을 맞춰 입력 |
| 통신 도중 노드가 멈춤 (데드락) | 콜백 함수 내부에서 다른 동기 통신을 중첩 대기함 | 비동기 호출(`call_async`)과 완료 콜백 함수 등록 방식으로 구조 개선 |

## 연습문제

1. `AddTwoInts` 타입을 그대로 사용하되, 두 수의 곱을 응답하는 곱셈 서비스를 구현하고 수동 호출 테스트를 실행하시오.
2. 클라이언트 노드가 실행될 때 고정 인자(3, 5) 대신 프로그램 실행 인수(`sys.argv`)로 두 숫자를 입력받아 서비스 요청을 수행하도록 코드를 고쳐 보시오.
3. `ros2 service call` 명령어를 통해 곱셈 서비스 서버를 호출해 결과를 확인하시오.

## 정리

* 서비스는 클라이언트가 요청하고 서버가 이에 상응하는 결과를 반환하는 일대일 양방향 요청-응답 통신 모델입니다.
* 서비스 타입(`.srv` 파일)은 `---` 기호를 사용해 윗칸의 요청(Request) 데이터 필드와 아래칸의 응답(Response) 데이터 필드를 나눕니다.
* 서비스 노드를 설계할 때 동기식 대기 루프는 교착 상태(deadlock)를 일으키기 쉬우므로, 비동기 호출(`call_async`)과 완료 콜백 함수 구조를 결합해 구현하는 것이 중요합니다.
* `ros2 service call` 등 전용 디버깅 CLI 명령을 활용하면 노드 생성 전이나 개발 과정에서 손쉽게 연산 검증이 가능합니다.

---

## 핵심 용어

* **서비스(service)**: 요청과 응답 구조를 갖는 일대일 양방향 통신 방식
* **DDS(Data Distribution Service)**: 마스터 서버 없이 분산 디스커버리를 제공하는 분산 통신 규격
* **RMW(ROS Middleware)**: ROS 2와 하부 DDS 구현체 간 통신을 중개하여 상호 이식성을 높이는 미들웨어 추상화 계층
* **비동기 호출(call_async)**: 요청을 보낸 뒤 응답이 올 때까지 프로그램 실행을 멈추지 않고, 응답 수신 시 비동기로 콜백을 수행하는 통신 패턴
* **LTS(Long-Term Support)**: 5년의 장기 기술 지원을 보장하는 ROS 배포판 버전 규격 (예: Jazzy Jalisco)

## 연습문제 해설(요약)

* **1번 문제**: `MultiplyServer`를 생성하여 콜백 로직에 `response.sum = request.a * request.b`를 설정합니다.
* **2번 문제**: Python의 `sys` 패키지를 가져와 `main` 혹은 초기화 함수 내에서 `a = int(sys.argv[1])`, `b = int(sys.argv[2])` 형태로 입력을 받아 처리합니다. (`ros2 run my_py_pkg client_node 4 9` 방식으로 기동)
* **3번 문제**: `ros2 service call /multiply example_interfaces/srv/AddTwoInts "{a: 5, b: 6}"` 명령을 사용해 `sum=30` 결과를 검증합니다.
* **4번 문제**: (가) 계속 흐르므로 **토픽**, (나) 속도 명령 지속 전송이므로 **토픽**, (다) 1회성 변경 요구이므로 **서비스**, (라) 이미지 데이터 스트리밍이므로 **토픽**, (마) 오래 걸리는 연속 작업과 중간 모니터링이 필요하므로 **액션**.
